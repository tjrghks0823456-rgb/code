# 시스템 구성

이 프로젝트는 Windows Forms 화면 계층, 장비 제어 계층, 공정 로직 계층, 데이터 계층으로 나누어 구성되어 있습니다.

## 전체 흐름

```text
사용자 조작
  ↓
MainForm / DeviceControlForm
  ↓
Controls / Monitor
  ↓
Logic / System
  ↓
EtherCAT Controller / IEG3268_Dll.dll
  ↓
실제 장비 입출력
```

## 계층 설명

### 화면 계층

`MainForm`, `DeviceControlForm`, `LoginForm`, `UserManagementForm`처럼 사용자가 직접 조작하는 화면입니다. 버튼, 상태 표시, 목록, 모니터링 패널을 통해 장비 상태를 확인하고 제어 명령을 전달합니다.

### 컨트롤 계층

`Controls` 폴더는 챔버, FOUP, 로봇 암, 상태 패널처럼 재사용 가능한 UI 구성 요소를 담고 있습니다. 메인 화면은 이 컨트롤들을 조합해서 장비 전체 레이아웃을 구성합니다.

### 공정 로직 계층

`Logic` 폴더는 챔버 공정 시간, 웨이퍼 이동 순서, 공정 시퀀스, 파이프라인 시뮬레이션을 담당합니다. UI에서 시작 명령을 받으면 공정 흐름을 계산하고 각 장비 동작 순서를 제어합니다.

### 장비 제어 계층

`System` 폴더는 EtherCAT 연결, IO 매핑, 챔버 IO, 경광등 IO 같은 실제 장비 입출력과 가까운 코드를 담고 있습니다. 장비 DLL 호출을 한 곳에 모아 UI 코드와 하드웨어 제어 코드를 분리합니다.

### 데이터 계층

`Data` 폴더는 레시피 저장, 조회, 수정, 데이터베이스 초기화를 담당합니다. 레시피 편집 화면과 챔버 런타임에서 같은 저장소 로직을 사용합니다.

## 설계 의도

- 화면 코드와 장비 제어 코드를 분리해서 유지보수를 쉽게 합니다.
- 공정 시나리오를 별도 로직 클래스로 분리해 테스트와 수정이 쉽도록 합니다.
- 챔버, FOUP, 로봇 같은 장비 요소를 패널 단위로 구성해 메인 화면이 복잡해지는 것을 줄입니다.
- 실제 장비 연결 전에도 시뮬레이션 흐름을 확인할 수 있도록 모니터링 로직을 포함합니다.
