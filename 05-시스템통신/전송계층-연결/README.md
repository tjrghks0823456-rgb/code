# 전송계층 연결 예제 정리

이 저장소에는 전송 계층에서 자주 비교하는 UDP 클라이언트 예제와 TCP 클라이언트 예제가 함께 들어 있습니다.

## 한눈에 보기

| 구분 | 위치 | 방식 | 핵심 특징 |
| --- | --- | --- | --- |
| UDP 예제 | `ConsoleApp3/ConsoleApp3/Program.cs` | `SocketType.Dgram`, `ProtocolType.Udp` | 연결을 먼저 맺지 않고 서버 주소로 데이터를 보냅니다. 응답이 없을 수 있어서 제한 시간 처리가 있습니다. |
| TCP 예제 | `TCP/TCP/Program.cs` | `SocketType.Stream`, `ProtocolType.Tcp` | `Connect()`로 서버와 연결한 뒤 데이터를 순서대로 보냅니다. |

## UDP 클라이언트

UDP 예제는 `ConsoleApp3` 폴더에 있습니다. 프로젝트 이름은 `ConsoleApp3`이지만, 코드의 네임스페이스와 소켓 설정을 보면 UDP 클라이언트입니다.

확인 포인트:

- `SocketType.Dgram`은 데이터그램 방식입니다.
- `ProtocolType.Udp`를 사용합니다.
- `SendTo()`로 목적지 주소에 데이터를 보냅니다.
- `ReceiveFrom()`으로 응답을 받습니다.
- 연결을 유지하지 않기 때문에 서버 응답이 없을 수 있어 `ReceiveTimeout`을 사용합니다.

## TCP 클라이언트

TCP 예제는 `TCP` 폴더에 있습니다. 서버에 먼저 접속한 다음 정해진 문자열들을 50바이트 크기로 맞춰 전송합니다.

확인 포인트:

- `SocketType.Stream`은 스트림 방식입니다.
- `ProtocolType.Tcp`를 사용합니다.
- `Connect()`로 서버와 연결합니다.
- `Send()`로 연결된 서버에 데이터를 보냅니다.
- 연결 기반이라 데이터 순서와 전달 안정성을 UDP보다 더 신경 써 줍니다.

## TCP와 UDP 차이

| 비교 | TCP | UDP |
| --- | --- | --- |
| 연결 방식 | 먼저 연결함 | 연결 없이 보냄 |
| 코드에서 보이는 차이 | `Connect()`, `Send()` | `SendTo()`, `ReceiveFrom()` |
| 신뢰성 | 높음. 순서와 전달을 관리함 | 낮음. 손실 가능성이 있음 |
| 속도와 부담 | 상대적으로 무거움 | 상대적으로 가벼움 |
| 어울리는 상황 | 채팅, 파일 전송, 로그인처럼 정확성이 중요한 경우 | 게임 위치 정보, 방송, 실시간 센서처럼 빠른 전송이 중요한 경우 |

## 실행 전 확인할 것

두 예제 모두 기본 서버 주소와 포트가 아래처럼 맞춰져 있습니다.

- 서버 IP: `172.16.11.92`
- 서버 포트: `9000`

다른 서버로 테스트하려면 실행할 때 IP 주소를 인수로 넘기거나, 코드의 `SERVERIP` 값을 바꾸면 됩니다.
