# 반도체 AI 장비제어 학습 프로젝트

이 저장소는 반도체, AI, 장비제어, 데이터 분석, GUI, 클라우드 운영 관련 과목에서 진행하는 실습과 프로젝트를 정리하기 위한 공간입니다.

아직 과목별 프로젝트가 모두 정해진 상태가 아니기 때문에, 처음부터 하나의 완성 프로젝트로 고정하지 않고 여러 과목의 프로젝트를 담을 수 있는 큰 분류 기준을 먼저 잡았습니다.

## 전체 방향

이 저장소의 목표는 다음과 같습니다.

> 반도체 장비 환경을 이해하고, 프로그래밍을 중심으로 장비 제어, 데이터 수집, AI 분석, GUI 화면, 클라우드 운영까지 단계적으로 확장하는 학습형 프로젝트 저장소

즉, 단순히 코드만 모으는 저장소가 아니라 과목별 실습과 프로젝트를 연결해서 하나의 포트폴리오로 발전시키는 것을 목표로 합니다.

## 큰 분류 기준

```text
code/
├─ 01-프로그래밍기초/
├─ 02-AI프로그래밍/
├─ 03-장비제어/
├─ 04-데이터분석/
├─ 05-시스템통신/
├─ 06-GUI화면/
├─ 07-클라우드운영/
├─ 08-품질관리/
├─ 09-반도체공정/
├─ 공통자료/
└─ 문서/
```

## 폴더별 역할

### 01-프로그래밍기초

프로그래밍 기초 과목과 기본 코딩 실습을 정리하는 공간입니다.

- Python 기초 문법
- 조건문, 반복문, 함수
- 클래스와 객체
- 파일 입출력
- 간단한 알고리즘 문제

관련 과목:

- 프로그래밍실습
- AI프로그래밍기초

### 02-AI프로그래밍

AI 관련 실습과 모델 개발 프로젝트를 정리하는 공간입니다.

- 머신러닝 기초
- 센서 데이터 기반 이상 탐지
- 장비 상태 예측
- 공정 데이터 분석 모델
- AI 제어 실습 코드

관련 과목:

- AI소프트웨어 활용 및 코딩
- AI제어실습
- AI프로그램심화

### 03-장비제어

장비 제어와 로봇 제어 관련 프로젝트를 정리하는 공간입니다.

- 센서값 기반 장비 제어
- 모터 제어
- PID 제어
- 장비 상태 관리
- 로봇팔 제어 시뮬레이션

관련 과목:

- 장비제어기초
- 장비제어심화
- 장비제어알고리즘
- 장비로봇제어

### 04-데이터분석

데이터 분석과 시각화 프로젝트를 정리하는 공간입니다.

- CSV 데이터 분석
- 센서 로그 분석
- 공정 데이터 시각화
- 통계 분석
- 빅데이터 처리 기초

관련 과목:

- 빅데이터분석기초
- 빅데이터활용

### 05-시스템통신

운영체제, 네트워크, 시스템 설계, 장비 통신 관련 프로젝트를 정리하는 공간입니다.

- TCP/UDP 통신
- MQTT 기반 IoT 통신
- 장비 데이터 송수신
- 시스템 구조 설계
- 프로세스와 스레드 실습

관련 과목:

- 운영체제
- 시스템분석및설계
- 장비통신SW개발
- 장비IoT센서제어

### 06-GUI화면

사용자 화면, GUI, 웹 대시보드 관련 프로젝트를 정리하는 공간입니다.

- 장비 상태 모니터링 화면
- 센서 데이터 그래프
- 제어 버튼 UI
- 관리자 화면
- 웹 기반 대시보드

관련 과목:

- 장비GUI제어
- 프로그램실습
- 직업과경력개발

### 07-클라우드운영

클라우드 운영과 배포 관련 실습을 정리하는 공간입니다.

- Docker 기초
- 서버 배포
- API 운영
- 클라우드 환경 설정
- 로그 관리

관련 과목:

- 클라우드운영실습

### 08-품질관리

소프트웨어 품질관리와 테스트 관련 내용을 정리하는 공간입니다.

- 테스트 코드
- 버그 리포트
- 코드 리뷰 기록

- 장비 개론 정리

## 과목별 프로젝트 관리 방식

각 과목에서 프로젝트가 생기면 해당 분류 폴더 안에 새로운 프로젝트 폴더를 만듭니다.

예시:

```text
03-장비제어/
└─ 모터제어기초/
   ├─ README.md
   ├─ src/
   └─ tests/
```

데이터 분석 프로젝트라면 다음처럼 관리할 수 있습니다.

```text
04-데이터분석/
└─ 센서로그분석/
   ├─ README.md
   ├─ data/
   ├─ notebooks/
   └─ src/
```

## 최종 확장 방향

처음에는 과목별 실습을 따로 정리하지만, 나중에는 여러 과목 내용을 연결해서 하나의 종합 프로젝트로 확장할 수 있습니다.

최종 목표 예시:

> 가상 반도체 장비에서 발생하는 센서 데이터를 수집하고, AI로 이상 여부를 판단하며, GUI 화면에서 장비 상태를 확인하고 제어하는 시스템

이 구조를 기준으로 학기별 프로젝트가 생길 때마다 폴더를 추가하고, 각 프로젝트의 README를 작성하면서 포트폴리오 형태로 발전시킵니다.
